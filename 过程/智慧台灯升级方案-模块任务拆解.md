## 模块4：华为云 IoT 对接

**难度**：★★★★★ ⚠️ 最复杂

**目标**：实现设备上云，支持远程监控和控制

**任务清单**：
- [ ] WiFi 网络连接：确保开发板能联网
- [ ] MQTT 客户端：集成 MQTT 库（lwip 已有，或用 paho mqtt）
- [ ] 华为云认证：设备 ID + 密钥方式连接 IoTDA Broker
- [ ] 数据上报：定时上报灯光状态和亮度到华为云
- [ ] 指令接收：订阅下行 Topic，接收云端开关/调光指令
- [ ] JSON 数据模板：定义产品模型（属性：light_switch、brightness、lux_value）

**涉及技术**：
- MQTT 协议（端口 1883 或 8883 TLS）
- 华为云 IoTDA 平台操作
- lwip 网络栈
- JSON 解析（cJSON）

**需要人工配合**：
- 注册华为云账号，开通 IoTDA 服务
- 创建产品和设备，获取设备 ID、密钥
- 配置 MQTT Topic 和数据模板
- 配置开发板 WiFi 连接

**备注**：大哥之前有华为云 MQTT 经验，捡起来会快。核心流程：连接 Broker → 订阅 Topic → 发布数据 → 接收指令。

## 建议开发顺序

```
模块1（硬件驱动）→ 模块2（HDF扩展）→ 模块3（HAP应用）→ 模块5（集成）→ 模块4（华为云）
```

**理由**：先完成本地功能（模块1-3），确保台灯能本地控制和调光；再做云端对接（模块4），这样即使云端部分卡住，本地功能也是完整可用的。

**当前进度**：模块1、2、3 已完成，下一步是模块4（华为云 IoT 对接）或模块5（系统集成测试）。
